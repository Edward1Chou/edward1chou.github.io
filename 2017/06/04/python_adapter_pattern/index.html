python适配器模式-----结构型模式(1)

发表于 2017-06-04 | 分类于 design_patterns

结构型设计模式

结构型设计模式处理一个系统中不同实体（比如，类和对象）之间的关系，关注的是提供一种简单的对象组合方式来创造新功能。

适配器模式(Adapter pattern)

它是一种结构型设计模式，帮助我们实现两个不兼容接口之间的兼容。

不兼容接口含义

我们希望把一个老组件用于一个新系统中，或者把一个新组件用于一个老系统中，不对代码做修改两者就能通信的情况是很少见的。

但又不是总能修改代码的，或因为我们无法访问这些代码（例如，组件以外部库的方式提供），或因为修改代码本身就不切实际。

在这种情况下，我们可以写一个额外的代码层，该代码层包含让两个接口之间通信需要进行的所有修改。这个代码层叫做适配器

举例

电子商务系统订单总价

要增加货币的结算方式，如果我们又系统源码，可以扩展系统，方法是添加一些新函数，将金额进行转换。但如果应用仅以外部库的方式提供，我们无法访问其源代码，解决方案是编写一个包装器（又叫适配器）将数据从给定的格式转换成期望的格式。

使用范围，比如：

  • 数据转换；
  • 函数转换：想使用一个接口，期望它是function_a()，但仅有function_b()可用，那么可以使用一个适配器进行适配；
  • 函数参数：有一个函数要求参数x,y,z，但是我们只有一个带参数x,y的函数。

现实生活中的例子：各种充电器的插头适配器

软件中的例子：

  • Grok是python的一个Web框架，专注于敏捷开发.Grok使用适配器，让已有对象无需变更就能符合指定API的标准。
  • python的第三方包Traits也使用了适配器模式，将没有实现某个指定接口（或一组接口）的对象转换成实现了接口的对象。

开放/封闭原则(open/close principle)

它是面向对象设计的基础原则之一（SOLID中的O），声明一个软件实体应该对扩展是开放的，对修改是封闭的。本质上这意味着我们应该无需修改一个软件实体的源代码就能拓展其行为。所以，适配器模式遵从开放/封闭原则

因此，在某个产品制作出来之后，需要应对新的需求之时，如果希望其仍然有效，使用适配器是一种更好的方式。
