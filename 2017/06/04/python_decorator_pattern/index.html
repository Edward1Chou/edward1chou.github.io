python修饰器模式-----结构型模式(2)

发表于 2017-06-04 | 分类于 design_patterns

引言

我们想要对一个对象添加额外的功能，都有下面这些不同的可选方法。

  • 添加一个新方法（如果合理，可以直接将功能添加到对象所属的类）
  • 使用组合（在类中调用其他类，比如：请外人来帮忙）
  • 使用继承（继承父类，比如：老爸来帮忙）
  • 修饰器（抽取大量与函数功能本身无关的雷同代码到装饰器中重用）

与继承相比，通常优先选择组合，因为继承使得代码更难复用，继承关系是静态的，并且应用于整个类以及这个类的所有实例。

修饰器支持动态地（运行时）拓展一个对象的功能。修饰器（Decorator）模式能够以透明的方式（不会影响其他对象）动态地将功能添加到一个对象中。

在许多编程语言中，使用子类化（继承）来实现修饰器模式。在Python中，我们只要使用内置的修饰器特性就好了。

从实现角度来说，python修饰器是一个可调用对象（函数、方法、类），接受一个函数对象fin作为输入，并返回另一个函数对象fout。之前我们已经使用property修饰器将一个方法变现为一个变量。之后我们会学习如何实现自己的修饰器

软件中例子

应用案例

用于实现横切关注点（一般来说，应用中有些部件是通用的，可应用于其他软件，这样的部件被看作是横切关注点）如：

  • 数据校验
  • 事务处理（类似于数据库事务，意味着要么所有步骤都成功完成，要么事务失败）
  • 缓存
  • 日志
  • 监控
  • 调试
  • 业务规则
  • 压缩
  • 加密

使用修饰器模式的另一个常见的例子是图形用户界面（Graphical User Interface, GUI）工具集。在一个个GUI工具集中，我们希望将一些特性，比如边框、阴影、颜色以及滚屏，添加到单个组件/部件。
